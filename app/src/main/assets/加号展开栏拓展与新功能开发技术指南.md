# 加号展开栏拓展与新功能开发技术指南

本手册是“叙事诗小手机”系统中**微信聊天应用（win-chat）底部加号展开栏（#chat-expand-panel）**增设全新交互按钮与功能模块的通用规范指南。当需要引入新功能时，请严格遵守本手册所述的系统架构、文件流向和技术约束，以防出现组件穿透、离线缓存死锁或层级错乱等问题。

---

## 1. 核心架构与必改文件清单

在聊天底栏展开项中新增一个功能，通常涉及以下文件的联动修改。请指示 AI 助手针对这些文件提供精确的 `Search` 与 `Replace` 代码段：

### 1.1 `index.html`：UI 骨架挂载
*   **按钮挂载**：在 `#chat-expand-panel .expand-slider` 的对应页数 `.expand-page` 中加入新按钮（SVG 格式图标，**严禁使用 Emoji**，配置唯一的 `id`，例如 `id="btn-chat-coin"`）。
    *   *排版提示*：面板第 1 页已满（8个图标分占 2 行格栅），新功能默认依次排在第 2 页。
*   **弹出层/模态窗挂载**：
    *   **【极重要隔离原则】**：为了彻底杜绝由于层叠上下文混乱而导致的“点击穿透”、“组件遮挡”或“指针事件（Pointer Events）失效”，**所有与新功能相关的绝对定位遮罩或弹出层（Modal/Overlay），必须在物理上作为子节点（Nesting）套袋收纳在 `#win-chat` 应用窗口内部**，禁止写在全局 `body` 的根目录下。
*   **脚本导入**：在 `index.html` 尾部引入为新功能独立编写的控制脚本（例如 `<script src="app_chat_coin.js"></script>`）。

### 1.2 `sw.js`：PWA 离线资源注册与强更新
*   **资源追加**：在新功能涉及专属 CSS 或 JS 脚本时，必须将该相对路径（如 `./app_chat_coin.js`）手动追加至 `sw.js` 内的 `ASSETS` 静态资源缓存数组中，确保离线状态下应用正常开启。
*   **缓存版本号强刷**：必须将 `CACHE_NAME` 版本号手动向上递增一级（例如从 `v12` 升级至 `v13`），强制让浏览器注销旧缓存并加载包含新 HTML 骨架的代码，否则由于 Cache-First 策略，用户端将一直卡在旧版界面中。

### 1.3 `app_prompts.js`：大模型提示词中枢注入 (如需影响AI回复)
*   若该功能需要大模型（AI角色）在对话中受到规则或设想约束（例如“剧情走向要求”或“硬币正反面反应”）：
    *   在 `buildGlobalSystemPrompt(sessionId)` 拼装编译链中，获取 `sessions` 会话配置中的对应新属性。
    *   通过特定的 `depth`（深度层级，例如 `-480` 左右，介于微信规则与时间感知之间）将最新的指令加入 `segments` 数组中。

### 1.4 `app_chat.js`：对话主程序路由适配
*   在 `app_chat.js` 底部的 `bindMultimediaEvents()` 或相关事件绑定区，注册新按钮的点击监听，用于关闭加号面板并唤起您的新功能：
    ```javascript
    const btnNewFeature = document.getElementById("btn-chat-new-feature");
    if (btnNewFeature) {
      btnNewFeature.onclick = () => {
        document.getElementById("chat-expand-panel").classList.remove("active");
        if (window.newFeatureSystem) window.newFeatureSystem.openModal();
      };
    }
    ```

### 1.5 专属控制脚本：`app_chat_xxx.js`（全新独立创建）
*   **闭环设计**：为新功能创建单文件 JS 控制脚本，负责其数据读取、模态窗切换、API 交互等。
*   **【自注册事件设计】**：为了防范由于 PWA 加载顺序、异步脚本延迟或 `app_chat.js` 冷启动执行锁导致事件丢失，**新脚本内必须编写防御性自注册函数**。在脚本解析完毕时，自动搜寻按钮并绑定点击事件：
    ```javascript
    function bindFeatureTrigger() {
      const btn = document.getElementById("btn-chat-new-feature");
      if (btn) {
        btn.onclick = (e) => {
          if (e) e.preventDefault();
          document.getElementById("chat-expand-panel").classList.remove("active");
          newFeatureSystem.openModal();
        };
      }
    }
    if (document.readyState === "loading") {
      document.addEventListener("DOMContentLoaded", bindFeatureTrigger);
    } else {
      bindFeatureTrigger();
    }
    ```

## 2. 数据库设计与持久化防崩规范

### 2.1 临时/持久非索引字段的存储 (Dexie.js 优化)
*   如果新功能只需要记录某个会话（Session）的某种状态、文本或开关设置：
    *   **无需在 `db.js` 中声明新字段或升级数据库版本**！Dexie.js 允许在记录中存储任意非 indexed 的属性。
    *   可以直接调用 `db.sessions.update(activeSessionId, { newProperty: value })` 将其写入对应的会话配置中。
*   *注意*：如果是需要物理建表（如 HTML 互动卡片表 `html_cards`），则必须在 `db.js` 中新增表，并在 `app_settings.js` 中做好配套管理。

### 2.2 格式化、备份与导入防死锁 (设置页适配)
*   **【关键警示】**：如果您为了新功能**新增了 IndexedDB 数据库表**，必须同时在 `app_settings.js` 的以下 4 个位置进行手动适配，否则备份与还原功能将直接死锁报错，甚至导致全手机数据格式化假死：
    1.  `computeStorageUsage()`：在统筹总记录数及 `fullDataObj` 文件物理体积计算中，加入新表数据的 `toArray()`。
    2.  `clearAllAppData()`：全格式化清空时，追加 `await db.new_table.clear();` 清空操作。
    3.  `exportBackup()`：在导出大 JSON 文件的 `rawBackup` 结构体中，追加新表的导出：`new_table: await db.new_table.toArray()`。
    4.  `importBackup()`：**最重要的一步**。在 `db.transaction('rw', [...], async () => { ... })` 读写事务锁声明数组中，**必须将新表（`db.new_table`）加入事务 RW 锁列表**，并在事务块中执行清除与批量写入 `bulkAdd()`。不加事务锁强行写入会引发事务跨域冲突假死。

## 3. UI 视觉与交互规范

*   **色调体系**：采用系统的“温和现代浅色”或“暗黑护眼石墨灰（Deep Slate & Indigo）”色调。避免刺眼的亮蓝或亮青，整体保持温润、低饱和度的磨砂感。
*   **去 Emoji 指令**：严禁使用 Emoji 字符作为图标。所有图标均需使用高清晰度的纯手绘 **SVG 路径（Feather / Lucide 风格矢量图）** 拼装。
*   **横向拖拽与圆点导航切换**：底栏展开面板的滑动组件采用 `scroll-snap-type: x mandatory`。由于 PC 浏览器上无法直接用普通鼠标左键进行按住横向拖拽滑动，**必须确保展开面板底部的滚动圆点（`.dot`）拥有点击滑动导航事件**：
    ```javascript
    dots.forEach((dot, idx) => {
      dot.style.cursor = "pointer";
      dot.onclick = () => {
        slider.scrollTo({ left: slider.clientWidth * idx, behavior: "smooth" });
      };
    });
    ```
