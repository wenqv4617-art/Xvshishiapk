### 2. 新增核心函数与逻辑解说

#### 2.1 财务隔离
*   `getBalanceKey()` / `getLedgerKey()`：根据当前 `localStorage` 中的 `active_me_id` 为每个用户人设动态计算出隔离的专属 LocalStorage 键名，从而实现在切换面具时余额与账单流水的独立变更。

#### 2.2 朋友圈手势交互
*   `startCommentLongPress(el, commentId, event)`：挂载触控和鼠标的长按（800ms）监听，触发评论行背景变红高亮并显示垃圾桶 SVG，且通过 Pointer Events 样式规则阻断了 SVG 内部 Path 元素对点击事件的干扰。
*   `handleCommentClick(momentId, commentId, commenterName, event)`：对长按状态进行检查。如果已被激活，点击行为将安全折叠恢复原状而不再触发回复模态窗，解耦了删除状态与回复状态。

#### 2.3 仿真时间与流逝计算
*   `getMessageDisplayDate(msg, sess)`：在渲染每条消息时调用。若单聊设置中关闭了“时间感知”，系统将根据虚拟起始时间（`customTimeData`）和设定保存时戳（`customTimeSavedAt`）累加该消息发送时的相对物理差额，计算出拟真场景下的显示时间。
*   `formatWeChatTime(date, relativeToDate)`：实现微信拟真的多态时间表示。根据传入的目标时间与相对时间（Simulated Now）的对比，按天、昨天、当年、跨年等不同格式渲染出诸如 `12:00`、`昨天 15:30` 或 `2026年1月1日 12:00` 样式的灰色系统时间戳。

#### 2.4 多物理手势解耦与防抖
*   `safeOpenMomentFromShare(momentId, event)`：为朋友圈转发卡片引入了 500 毫秒延时防抖。若用户在 0.5 秒内双击消息气泡呼出操作菜单，定时器将自动在双击事件中被清空清除，避免了“单击跳转朋友圈”与“双击呼出编辑菜单”之间的事件抢占冲突。

#### 2.5 消息撤回状态机
*   `toggleRecallContent(el, event)`：撤回的消息将转化为完全居中、去 bubble 耦合的灰色系统通知条。点击“查看”按钮可就地向下展开/收起已被撤回的原始消息本体，避免了系统 Alert 弹窗对用户操作的打扰。

#### 2.6 长按 Emoji 贴纸反应
*   `startBubbleLongPress(msgId, bubbleEl, e)`：检测气泡长按达 2 秒触发。内部会动态抓取当前会话最近的 20 条消息（即 10 轮交互），确保该消息在 20 条区间内才予以激活。
*   `saveReaction(msgId, emoji)` / `removeReaction(msgId, event)`：提供表情反应贴纸的覆盖和一键点击剥除功能，并在剥除时进行 `stopPropagation` 阻止底层气泡事件冒泡。

### 3. 技术路线与实现架构分析

```
                        ┌──> [app_prompts.js] (Prompt 拼装与动态条件注入)
                        ├──> [app_chat.js] (时间随动计算、长按/双击手势及状态变更)
[用户交互 / 大模型反应] ────┼──> [app_moments.js] (评论长按机制、解耦同组反应范围)
                        ├──> [app_chat_quote.js] (滑动手势与选择器解耦判定)
                        └──> [app_wallet.js] (人设随动钱包余额多态存储)
```

#### 3.1 微信多态时间推算技术路线
为了在单聊中完全支持“关闭时间感知（使用设定时间）”和“开启时间感知（跟随真实时钟）”两种模式，系统设计了 **相对流逝比例推演机制**：
*   当关闭时间感知时，系统并不使用当前真实的系统时钟作为显示时戳，而是通过计算 `Date.now() - sess.customTimeSavedAt` 得到该对话在现实世界中所流逝的毫秒差额。
*   将该差额直接累加到设定的基准时间（如 2026-01-01 12:00）上，得出一个随真实时空 1:1 正常流逝演进的虚拟场景时间。
*   主列表（Session List）的排序则永远依赖消息的真实物理时戳，从而保障了会话列表检索性能与聊天视觉场景体验的统一。

#### 3.2 自带事件阻断的绝对定位 UI 架构
为了在不使用额外第三方库的情况下，实现气泡长按 Emoji 选择器、气泡原地撤回展开、已贴表情点击移除等复杂手势，系统贯彻了**作用域物理隔离设计**：
*   撤销了在 `.msg-bubble` 外层套用 `.msg-content-wrapper`（具有 display: flex）的设计。转而让具体气泡卡片本身承担 `position: relative`，从而保留了全局 `style.css` 和 `chat.css` 的所有原生 padding、margin 和 avatar 间距属性，消除了气泡对折缩窄和间距紊乱的异常。
*   利用 `e.target.closest` 技术对 Quotation 向左滑动进行始发点探测。如果触控手势是从 `.bubble-emoji-picker` 或 `.bubble-attached-emoji` 内部发起的，滑动引用模块会在初始化阶段予以直接拦截并 `return`，从而彻底解决了表情选择器横向滑动选择与气泡左滑引用之间的冲突。

本日志记录了今日对“叙事诗小手机”系统进行的系列功能扩充、体验升级与代码纠偏。此次迭代重点围绕“沉浸式拟真体验”与“操作流畅度”，通过自研消息提示中枢、时序不退焦、请求中断网关以及拟真长按动效，基本消除了浏览器原生组件带来的跳出感，并修复了合并过程中由于拼写闭合导致的语法阻塞。

### 1. 更新功能板块与技术细节

#### 1.1 全局 PWA 自研对话框与消息组件系统 (Toast & Modal Dialog Engine)
*   **开发背景**：原生浏览器的 `alert()`、`confirm()` 以及 `prompt()` 灰色弹窗会严重破坏虚拟手机的扮演沉浸感 [1]。
*   **重构方案**：
    *   在 `app_chat.js` 头部主动注入了一套具有毛玻璃质感、微白色发光描边、深色半透明悬浮条的 CSS 与 JavaScript 双端联合组件。
    *   **接口封装**：
        *   `showToast(msg, duration)`: 在底部 120px 处呈现 0.2s 渐入淡出的半透明悬浮条 [1]。
        *   `showCustomAlert(title, message, callback)`: 全屏卡片式错误及结果反馈 Dialog [1]。
        *   `showCustomPrompt(title, defaultValue, callback)`: 卡片式数值输入 Dialog（替换充值、提现时粗暴的 `prompt` 录入）[1]。
        *   `showCustomConfirm(title, message, onConfirm)`: 带有“取消/确认”双通道的物理删除、归档确认卡片 [1]。
    *   **覆盖板块**：已无损重构并覆盖至**单聊专属设置保存**、**API 请求异常拦截**、**朋友圈动态转发/发送空内容提示**、**微信零钱充值与提现账单**、**朋友圈 AI 自发动态生成**、**档案库及 NPC 新建删除**、以及**深谈卡片回溯 Reroll**。

#### 1.2 时序不退焦发送与回车上屏系统 (Keyboard Focus & Enter Send)
*   **开发背景**：之前点击发送消息按钮时，输入框由于失去焦距（Blur）会导致移动端软键盘收起、弹回，影响多句连发的爽快感 [1]。
*   **重构方案**：
    *   **防抖不退焦**：在发送按钮上利用 `onmousedown` 的 `e.preventDefault()` 阻止输入框失去焦点 [1]；并在 `onclick` 上屏时，在逻辑尾部进行 `dialogInput.focus()` 的显式回焦兜底 [1]。
    *   **回车发送**：在“系统设置”的“桌面美化设置”中追加了 **`回车上屏 (Enter 发送)` 开关** [1]，状态利用 LocalStorage 持久化保存 [1]。在开启后，按 `Enter` 键会自动触发上屏且阻断默认换行，同样保持键盘不收回 [1]。

#### 1.3 大模型中断请求网关 (Request Abortion via AbortController)
*   **开发背景**：网络超时或模型推理过慢（特别是思考型长上下文推理）时，用户往往只能等待而无法干预，容易造成页面假死 [1]。
*   **重构方案**：
    *   引入了标准 `AbortController` 并在 `app_chat.js` 顶部声明了 `onlineAbortController` 与 `offlineAbortController` 中断控制量 [1]。
    *   **动作联动**：线上和线下发送请求后，“获取回复（✨）”按钮会切换成**浅红色的圆角停止方块图标** [1]；此时再次点击该停止按钮，将立即触发 `.abort()` 中断当前的 fetch 神经链接 [1]。
    *   **容错捕获**：在 catch 分支中精准识别 `AbortError` 并予以吞掉 [1]，不再报出长串的 HTTP 报错卡片，而是弹出一行温和的 Toast：「`当前请求已终止`」，同时瞬时将按钮图标和页头 typing 状态还原 [1]。

#### 1.4 拟真物理按压弹性回弹动效 (Two-stage Haptic Long Press)
*   **开发背景**：气泡长按唤出反应贴纸选择器时没有视觉过渡，反馈不够生动灵敏。
*   **重构方案**：
    *   利用两个并行的 `setTimeout` 构造了时间差：按压达到 **1.0秒（第一阶段）** 时，气泡上平滑附加 `.bubble-longpressing` 样式（`scale(0.95)` 微收缩）[1]；按压达到 **1.3秒（第二阶段）** 时，气泡瞬间复原尺寸，并唤出表情贴图选择器 [1]。
    *   中途松手、挪开指针等行为会调用 `cancelBubbleLongPress`，立即清除定时器并弹回原有尺寸，还原了类似于原生 iOS 触觉按压的深度回弹手感 [1]。

#### 1.5 表情反应面板选择器的物理对齐约束 (Reaction Picker Alignment)
*   **重构方案**：为了贴合微信/手 Q 在对话气泡上的视觉偏侧，调整了表情选择器的对齐方案。通过在 `showEmojiPicker` 内读取 bubble 节点的类名：如果为己方发言（`.self`），则选择器采用右对齐（`right: 0`）；若为对方发言（`.other`），则采用左对齐（`left: 0`）[1]。

#### 1.6 语法阻塞自愈修复
*   **问题排查**：在之前的组件合并过程中，头部多媒体场景卡片的 CSS 模板字符串反引号在中途被提早闭合了，导致后续的 `.pwa-toast` 等大段 CSS 裸码泄露至 JavaScript 全局执行域中，触发了 JS 语法解析器阻断 [1]。
*   **修复方案**：已重新将 `multimediaStyle.textContent` 修正为单一无缝的反引号字符串结构，保障了 `app_chat.js` 及其余协同组件的正常加载与评估 [1]。
